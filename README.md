# 🔐 ISMS-P 인증기준 RAG 챗봇

> ISMS-P 인증기준(101개)에 대해 자연어로 질문하면, 관련 기준을 검색해 **쉬운 말로 답하고 근거 기준 번호를 출처로 인용**하는 RAG 챗봇입니다.

**🔗 라이브 데모:** _(배포 후 URL 추가)_  ·  ⚠️ 비공식 참고용

---

## 🎯 해결하는 문제

ISMS-P(정보보호 및 개인정보보호 관리체계) 인증기준은 **3개 영역 101개**로 방대합니다. 실무에서는 이런 어려움이 있습니다.

- 어떤 상황이 **어느 기준에 해당하는지** 빠르게 찾기 어렵다
- 공식 안내서는 **법령체 문장**이라 한눈에 이해하기 어렵다
- 단순 키워드 검색은 **동의어·맥락**을 놓친다 (예: "접속 차단" ↔ "접근통제")

이 챗봇은 **의미 기반 검색(임베딩) + LLM 설명**으로, *"위험평가는 어느 기준이야?"* 같은 자연어 질문에 관련 기준을 찾아 **쉽게 설명하고 출처를 인용**합니다.

## ✨ 주요 기능

- 🔎 **의미 기반 검색** — 키워드가 정확히 일치하지 않아도 관련 기준을 찾음
- 💬 **출처 인용 답변** — 검색된 기준만 근거로 답하고 `[출처: 2.5.1]` 형식으로 표기
- 🛡️ **환각 억제** — 근거가 없으면 지어내지 않고 *"찾지 못했습니다"* 라고 답함
- 📚 **참고한 기준 원문 확인** — 답변 근거가 된 기준을 펼쳐서 직접 검증 가능
- ⚖️ **비공식 참고용 고지** — 공식 기준은 KISA 확인을 안내

## 🏗️ RAG 파이프라인

```
[인덱싱 — 1회]
  인증기준 101개(JSON)
    → 청크화 (기준 1개 = 청크 1개)
    → 한국어 임베딩
    → Chroma 벡터스토어 저장 (메타데이터: id/domain/category)

[질의 — 매 질문]
  사용자 질문
    → 질문 임베딩 → 벡터스토어 top-k(=4) 검색
    → 검색된 기준 + 질문을 Claude에 전달
    → "근거 기준만 사용, 출처 인용, 없으면 모른다고 답" 프롬프트
    → 답변 + 출처(기준 번호)
```

### 기술 스택

| 구분 | 사용 기술 | 비고 |
|------|-----------|------|
| 언어 | Python 3.11+ | |
| 임베딩 | `jhgan/ko-sroberta-multitask` | **한국어 특화**, 로컬 실행(무료) |
| 벡터 검색 | Chroma | 로컬, 코사인 유사도 |
| 답변 생성 | Anthropic Claude (Sonnet) | 출처 인용·환각 억제 프롬프트 |
| UI | Streamlit | |
| 배포 | Streamlit Community Cloud | 무료 |

> **한국어 임베딩 모델을 쓴 이유**: 인증기준 텍스트가 전부 한국어라, 다국어 모델보다 한국어 문장 유사도에 특화된 모델이 검색 품질에 유리합니다.

## 🚀 실행 방법

### 1) 환경 준비
```powershell
# 가상환경 생성 및 활성화 (Windows PowerShell)
python -m venv venv
.\venv\Scripts\Activate.ps1

# 의존성 설치
pip install -r requirements.txt
```

### 2) API 키 설정
`.env.example` 을 복사해 `.env` 를 만들고 키를 입력합니다.
```powershell
Copy-Item .env.example .env
# .env 파일을 열어 ANTHROPIC_API_KEY=sk-ant-... 입력
```

### 3) 데이터 검증 & 인덱스 빌드
```powershell
python load_data.py      # 101개 로드 및 필드 검증
python build_index.py    # 임베딩 → Chroma 인덱스 생성 (최초 1회 모델 다운로드)
```

### 4) 실행
```powershell
# 터미널에서 RAG 테스트
python rag.py "접근통제 관련 기준을 알려주세요"
python rag.py -r "개인정보 파기"   # 검색만 (API 불필요)

# 웹 앱 실행
streamlit run app.py
```

## 📌 면책

- 본 도구는 **비공식 참고용**입니다. 공식 인증기준은 반드시 [KISA ISMS-P](https://isms.kisa.or.kr)에서 확인하세요.
- 현재 기준(2023.11, 101개)을 사용합니다.
- 답변은 검색된 기준에 근거하며, LLM 특성상 오류가 있을 수 있습니다.

## 🛠️ 설계 포인트 (기술적 의사결정)

- **환각 억제 설계**: 검색된 기준만 근거로 사용하도록 프롬프트로 강제하고, 근거가 없으면 답을 지어내지 않고 "찾지 못했습니다"라고 응답
- **출처 인용**: 모든 답변에 근거 기준 번호를 표기해 사용자가 원문으로 검증 가능
- **graceful degradation**: API 호출이 실패해도(크레딧 부족 등) 검색 결과는 항상 표시되어 앱이 중단되지 않음
- **검색/생성 분리**: `retrieve()`는 API 없이 동작 → 비용 없이 검색 품질 테스트 가능
