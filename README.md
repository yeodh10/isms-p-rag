# 🔐 ISMS-P 인증기준 RAG 챗봇

> ISMS-P 인증기준(101개)에 대해 자연어로 질문하면, 관련 기준을 검색해 **쉬운 말로 답하고 근거 기준 번호를 출처로 인용**하는 RAG 챗봇입니다. 더불어 **회사 현황을 입력하면 분야별 기준 충족 여부를 AI가 보조 판단하는 "사전 자가점검(갭분석)" 모드**를 제공합니다.

**🔗 라이브 데모:** https://isms-p-rag-3lgqwqs9djupbpphpvsmdb.streamlit.app/  ·  ⚠️ 비공식 참고용

---

## 🎯 해결하는 문제

ISMS-P(정보보호 및 개인정보보호 관리체계) 인증기준은 **3개 영역 101개**로 방대합니다.

- 어떤 상황이 **어느 기준에 해당하는지** 빠르게 찾기 어렵다
- 공식 안내서는 **법령체 문장**이라 한눈에 이해하기 어렵다
- 단순 키워드 검색은 **동의어·맥락**을 놓친다 (예: "접속 차단" ↔ "접근통제")

이 챗봇은 **의미 기반 검색(임베딩) + LLM 설명**으로, *"위험평가는 어느 기준이야?"* 같은 자연어 질문에 관련 기준을 찾아 **쉽게 설명하고 출처를 인용**합니다.

## ✨ 주요 기능

- 🔎 **의미 기반 검색** — 키워드가 정확히 일치하지 않아도 관련 기준을 찾음
- 💬 **출처 인용 답변(스트리밍)** — 검색된 기준만 근거로 답하고 `[출처: 2.5.1]`로 표기
- 🛡️ **환각 억제** — 근거가 없으면 *"찾지 못했습니다"*; 검색 유사도가 임계값 미만인 무관 질의는 **LLM 호출 없이** 즉시 차단(비용 절감)
- 🗂️ **멀티턴 챗봇 UI** + 참고한 기준 원문 펼쳐보기
- 🩺 **사전 자가점검(AI 보조 갭분석)** — 분야를 고르고 회사 현황·증적을 입력(또는 **PDF·DOCX 업로드**)하면, **공식 주요 확인사항**을 근거로 기준별 **충족/부분충족/미흡**을 AI가 보조 판단하고 보완점을 제시. 점검한 분야를 **누적해 조직 전체 갭 리포트(.md)** 로 집계·다운로드 (ISMS 80 / ISMS-P 101 범위)
- 🔒 **공개 앱 보호** — 비밀번호 게이트 + 세션 레이트리밋(API 비용 남용 방지)

## 🖼️ 화면

**▶ [라이브 데모](https://isms-p-rag-3lgqwqs9djupbpphpvsmdb.streamlit.app/) 에서 직접 사용해 보세요** (사이드바에서 두 모드 전환):

- **💬 질문하기** — 자연어 질문 → 관련 기준 검색 → 출처 인용 답변(스트리밍) + 참고한 기준 펼치기
- **✅ 사전 자가점검** — 범위/분야 선택 → 현황 입력(또는 문서 업로드) → 기준별 충족/미흡 보조 판단 + 누적 갭 리포트

## 🏗️ RAG 파이프라인

```
[인덱싱 — 1회]
  인증기준 101개(JSON) → 한국어 임베딩 → Chroma 벡터스토어(메타데이터: id/domain/category)

[질의 — 매 질문]
  질문 → 임베딩 → top-k(=4) 검색
       → (최상위 유사도 < 임계값 이면 LLM 호출 없이 "찾지 못했습니다")
       → 검색된 기준 + 질문을 Claude에 전달
       → "근거 기준만 사용, 출처 인용, 없으면 모른다고 답" 프롬프트(스트리밍)
       → 답변 + 출처(기준 번호)
```

### 기술 스택

| 구분 | 사용 기술 |
|------|-----------|
| 언어 | Python 3.11+ |
| 임베딩 | `jhgan/ko-sroberta-multitask` (한국어 특화, 로컬 실행·무료) |
| 벡터 검색 | Chroma (코사인 유사도) |
| 답변 생성 | Anthropic Claude (`claude-sonnet-4-6`, 스트리밍) |
| UI | Streamlit |
| 배포 | Streamlit Community Cloud |
| 테스트 | pytest |

## 📊 검색 품질 (정량 평가)

3개 영역 전반의 **라벨된 질의 20개**로 측정 (`python eval_retrieval.py`):

| 지표 | 값 | 의미 |
|------|----|------|
| **Hit@1** | **95%** | 1순위가 정답 기준 |
| **Hit@4** | **100%** | top-4 안에 정답 포함 |
| **MRR** | **0.975** | 정답의 평균 역순위 |

검색 모델·파라미터를 바꿀 때 이 스크립트로 품질 회귀를 측정합니다.

## 🔒 배포 시 보안 / 비용 보호

공개 앱은 **소유자의 API 키로 Claude를 호출**합니다. 누구나 접속해 비용을 발생시킬 수 있으므로:

- **비밀번호 게이트**: Streamlit Secrets에 `APP_PASSWORD = "..."` 를 설정하면 접속 시 비밀번호를 요구합니다(미설정 시 공개).
- **세션 레이트리밋**: 세션당 시간 제한(기본 15회/시간) 내장. *(주의: 세션 단위라 분산 접근엔 한계 — 공개 시 비밀번호 게이트 병행 권장)*
- **키 관리**: API 키는 repo에 두지 말고 `.env`(로컬)·Secrets(배포)에만. `.gitignore`로 `.env` 제외.

## ⚠️ 알려진 한계 (정직하게)

- **집계/메타 질의 약점**: "관리체계는 몇 개 기준?" 같은 개수 질의는 답이 개별 기준 본문에 없어 RAG가 정직하게 *"찾지 못했습니다"*로 응답합니다(지어내지 않음).
- **요약문 ≠ 공식 원문**: 위 *데이터* 항목 참조.
- **콜드스타트**: Streamlit Cloud 무료 티어는 앱이 잠들면 깨어날 때 임베딩 모델(~440MB) 재다운로드로 첫 응답이 1~2분 걸릴 수 있습니다.

## 🛠️ 설계 포인트 (기술적 의사결정)

- **환각 억제**: 검색된 기준만 근거로 강제 + 근거 없으면 "찾지 못했습니다" + **저유사도 질의는 LLM 호출 전에 차단**(정확성 + 비용)
- **출처 인용**: 모든 답변에 근거 기준 번호 표기 → 사용자가 원문으로 검증 가능
- **신뢰성**: 데이터 제목·구조를 공식 KISA 안내서와 대조 검증, 검색 품질을 라벨셋으로 **정량 측정**
- **견고성**: 스트리밍 응답 + 타입드 예외 처리 + 타임아웃 + graceful degradation(API 실패해도 검색 결과는 표시)
- **운영**: 공개 배포 비용 보호(게이트·레이트리밋), 검색/생성 분리(비용 없이 검색 테스트), 버전 고정으로 재현성
